PeaZip is a free cross-platform file manager / archive manager software that provides unified portable GUI for many Open Source technologies as 7-Zip, FreeArc, PAQ, UPX... free alternative to WinRar, WinZip and similar proprietary software.
- Create 7Z, ARC, BZ2, GZ, *PAQ, PEA, QUAD/BALZ, TAR, UPX, WIM, XZ, ZIP files
- Open and extract ACE, ARJ, CAB, DMG, ISO, LHA, RAR, UDF, ZIPX files and more, over 200 archive types supported
Features of PeaZip includes: archives browser / explorer, integrated image viewer, batch creation and extraction of multiple archives at once, convert files, create self-extracting archives, split and join files, strong encryption with two factor authentication, encrypted password manager, secure deletion, find duplicate files, calculate hash and checksum, export job definition as script.

Archive browser / explorer and file manager


Screenshots showing PeaZip archive manager software creating a ZIP archive (showing file explorer in backgorund, first screenshot), and displaying file manager / archive manager browser GUI (second and third screenshots), in "Details" mode and "Icons and images" mode, working as explorer-like image viewer utility (last picture).

F6 toggles flat mode browsing, switching from explorer-like archive view to showing all content at same time, with full path displayed on the rightmost column of the file browser. Same file manager GUI is shown on Microsoft Windows and Linux systems.

When reading the content of an archive file, for most common formats (7z, cab, iso, jar, rar, tar, zip, zipx...) it is shown the archive (or package) structure tree on the left navigation area, for quick inspection of the content and for fast browsing to desired node.
To read the content of a spanned archive set, click on the first volume of the set (with extension 001, Z01, R01 or similar) and browse the archive as usual.

Formats like gz, bz2, xz are simply compressed file containers, they contains a single file compressed with Gzip, BZip2, LZMA algorithm respectively.
Some exceptions: only flat mode view is enabled for ace and arc formats; pea format does not show archive table of content (no content listing mode is currently implemented).





DARK MODE


PeaZip currently does not fully support Windows 10 dark mode, which is meant - by Microsoft design - for UWP applications only.
Unlike previous versions of Windows, applying Dark Mode theming does not changes system's colors values passed to Win32/Win64 applications, but changes only colors passed to UWP apps.
Until Windows 10 Dark Mode will be implemented by Microsoft in this bizarre way, purposely passing wrong values for on screen colors to non-UWP apps, supporting it will need each non-UWP application to replace the code for drawing each widget drawn by the system (e.g. checkboxes, combo boxes, listviews, etc), overriding standard - proven, stable, and optimized for speed - system drawing routines with custom ones.

There are however some possible workarounds in order to use PeaZip in dark mode.

1) Dark option: in Options > Settings > Theme it is possible to set the program's GUI to Dark, which will darken the colors within the limits to preserve legibility of system-drawn widgets (which cannot be drawn by the application in custom colors)


2) High Contrast: in Windows 10 system's settings it is possible to set on of the pre-set high contrast theme, which passes dark system colors (which can be further customized, creating custom dark themes) to all non-UWP applications, including PeaZip

Interactive file compression and decompression


First screenshots shows GUI for archive creation and file conversion (extraction and re-compression to new archive format), allowing to compress data to a single output archive or, optionally, compress separately each item in input list to create separate archive files at once.
Archiving GUI also features archive spanning (file split) option, in order to create multiple output volumes of maximum user-defined size.
Same screen allows to add and update files in existing archives for read & write supported formats such as 7z, arc, tar, wim, and zip archives.


Second screenshot displays archive extraction (decompression) GUI, allowing:
  • extraction of multiple archives at once, of same or mixed types, for all full supported or read-supported formats (like ace, rar, zipx etc), decompressing each entire archive
  • extraction of selected files decompressing selected content from a single archive of any read-supported format
In first case the input list contains the archives going to be decompressed, in the second case it will list the files and folders selected for extraction from the archives.

Both archiving and extraction screens features the option to delete original files after compression / decompression operation terminates, optionally using built-in secure data deletion feature.
Password (padlock icon) allows to create / extract encrypted archives.



Batch compression and decompression, scripting & scheduling


Console and Schedule tabs aims to bridge the gap from GUI to CL applications allowing exporting compression / decompression jobs defined in the interactive GUI (scripting) and, on Windows system, scheduling the execution to programmatically automate saved batch jobs.

Console tab is meant to support scripting with ease, automatically generating compression / decompression commands as defined in the interactive GUI and allowing to save and edit the task definition (in example for re-using the scripted task, or for syntax learning purpose) to batch .bat file or for any other type of scripts - UTF-8 encoding is supported.


Schedule tab allows scheduling saved task's execution (i.e. schedule backup and restore jobs) running the script file using system's scheduler to programmatically execute file compression and decompression tasks.
